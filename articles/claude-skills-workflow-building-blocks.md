## 「AIエージェントに毎回同じ説明をしていませんか?」

こんな経験はありませんか?

- 毎回Claudeに「このプロジェクトではTypeScriptのこの規約に従ってください」と説明している
- データ分析の手順を何度も同じように指示している
- ブランドガイドラインやコンプライアンス要件を会話のたびに伝えている

このような状況は、時間の無駄であり、説明の一貫性も保てません。

プロンプトを使えばある程度は解決できますが、プロンプトは1つの会話にしか適用されません。プロジェクトを使えば永続的なコンテキストを保持できますが、プロジェクトをまたいで知識を共有することはできません。

**Claude Skillsは、この問題を解決します。**

Skillsは、専門知識をポータブルな形でパッケージ化し、必要なときだけ動的にロードする仕組みです。これにより、会話やプロジェクトをまたいで専門知識を再利用できます。

## Claude Skillsとは

Claude Skillsは、特定のドメインに関する指示、スクリプト、リソースをまとめた専門知識フォルダーです。Claudeは関連性のあるSkillsを動的に発見し、ロードします。

### What（何をするのか）

Skillsは以下の3つの特徴を持っています:

1. **段階的な開示（Progressive Disclosure）**
   - メタデータ（約100トークン）が最初にロードされます
   - 必要なときだけ完全な指示（5,000トークン未満）がロードされます
   - トークンの無駄を削減し、効率的に動作します

2. **動的ロード（Dynamic Loading）**
   - 特定のタスクに関連するSkillsだけが有効化されます
   - 不要なSkillsはロードされません
   - コンテキストを節約しながら専門性を提供します

3. **再利用可能な専門知識（Reusable Expertise）**
   - 会話やプロジェクトをまたいで利用可能です
   - 一度定義すれば、どこでも使えます
   - チーム内で知識を共有できます

### Why（なぜ必要なのか）

Skillsが必要な理由は、以下の3つのユースケースにあります:

**1. 組織の手順やガイドライン**
- コンプライアンス要件
- ブランド標準
- 文書テンプレート

これらは組織全体で一貫性を保つ必要があり、毎回説明するのは非効率です。

**2. ドメイン固有の専門知識**
- データ分析の手法
- PDF操作のベストプラクティス
- 特定のフレームワークの使い方

これらは専門的な知識であり、詳細な指示が必要ですが、毎回プロンプトに含めるには長すぎます。

**3. 個人のワークフロー**
- ノート取りシステム
- コーディングパターン
- タスク管理の方法

これらは個人の効率性を高めるもので、環境に応じて自動的に適用されるべきです。

### How（どのように実装するのか）

Skillsは、特定のディレクトリ構造に配置されたMarkdownファイルとして実装されます。

基本的なSkillの構造:

```
.claude/
└── skills/
    └── excel-expert/
        ├── SKILL.md         # メタデータと指示
        ├── templates/       # テンプレートファイル
        └── examples/        # サンプルコード
```

`SKILL.md`の例:

```markdown
# Excel Expert Skill

## Description
Provides expertise in Excel data analysis and formula creation.

## When to Use
- Analyzing spreadsheet data
- Creating complex formulas
- Generating reports from Excel files

## Instructions
When working with Excel files:
1. Always validate data types
2. Use VLOOKUP for cross-referencing
3. Apply conditional formatting for insights
...
```

このように定義すると、Claudeはスプレッドシート関連のタスクを検出したときに自動的にこのSkillをロードします。

## Claudeエコシステムにおける位置づけ

Claudeには、Skillsの他にもいくつかのツールがあります。それぞれの使い分けを理解することが重要です。

### ツールの比較表

| ツール | 目的 | 永続性 |
|--------|------|--------|
| **Prompts** | 会話の指示 | 単一の会話 |
| **Projects** | 永続的なコンテキストワークスペース | プロジェクト内 |
| **Subagents** | 専門的なタスク委譲 | セッションベース |
| **Skills** | 再利用可能な専門知識 | すべての会話・プロジェクト |
| **MCP** | 外部ツールへの接続 | 継続的なアクセス |

### Skills vs MCP: 重要な違い

**Skillsは「方法」を教え、MCPは「アクセス」を提供します。**

具体例で考えてみましょう:

**Skillsの役割:**
「Excelレポートはこの数式を使ってフォーマットしてください」という知識を提供します。

**MCPの役割:**
実際のスプレッドシートファイルへのアクセスを提供します。

つまり、Skillsは「どのようにやるか」を教え、MCPは「何にアクセスするか」を可能にします。両者は補完的な関係にあります。

### Skills vs Subagents: 使い分け

**Skills:**
- どのエージェントでも適用できるポータブルな専門知識
- 例: 「データ分析の手法」「コーディング規約」

**Subagents:**
- 独立したワークフローで、専用のツール権限を持つ
- 例: 「特定のデータソースから情報を収集して分析する」タスク全体を委譲

Skillsは知識の提供、Subagentsはタスクの実行を担当します。

## 実践例: 複数ツールを組み合わせた研究エージェント

すべてのツールを組み合わせると、非常に強力なワークフローが実現できます。

研究エージェントの例:

```markdown
【構成要素】

1. **Project（プロジェクト）**
   - 研究テーマの背景知識を提供
   - 過去の会話履歴を保持

2. **MCP（ツール接続）**
   - Google Driveへのアクセス
   - GitHubリポジトリへのアクセス

3. **Skills（専門知識）**
   - 分析フレームワークの提供
   - レポート作成のテンプレート

4. **Subagents（専門エージェント）**
   - 文献調査を担当するエージェント
   - データ分析を担当するエージェント

5. **Prompts（会話指示）**
   - リアルタイムでリクエストを調整
```

### ワークフローの流れ

1. ユーザーが研究テーマについて質問
2. Projectから背景知識がロード
3. 分析Skillが自動的に有効化され、分析フレームワークを提供
4. MCPを通じてGoogle Driveから関連文書を取得
5. Subagentが文献調査を実行
6. レポート作成Skillのテンプレートを使って結果をまとめ
7. Promptで最終的な出力を調整

この統合アプローチは、個別のツールの能力を超えた強力なワークフローを実現します。

## やってみた結果

### 成功した点

私は実際にSkillsを活用して、以下のような成果を得ました:

- **時間削減:** 毎回同じ説明をする必要がなくなり、1タスクあたり平均5〜10分の時間を節約
- **一貫性の向上:** コーディング規約やドキュメント形式が自動的に適用され、品質が安定
- **知識の再利用:** 一度定義したSkillを複数のプロジェクトで活用でき、専門知識が蓄積

### 直面した課題と解決方法

**課題1: Skillsが多すぎるとどれが使われているか分からない**
- 解決方法: Skillのメタデータに明確な「When to Use」セクションを記述
- Claudeが適切なSkillを選択できるよう、具体的なトリガー条件を記載

**課題2: Skillの内容が古くなってしまう**
- 解決方法: 定期的にレビューする習慣を作る
- バージョン管理システム（Git）でSkillsを管理し、変更履歴を追跡

### 学んだこと

Skillsを効果的に使うためのポイント:

1. **具体的な条件を定義する**
   - 「いつこのSkillを使うか」を明確に記述
   - 抽象的な説明は避け、具体的なトリガーを記載

2. **5,000トークン以内に収める**
   - 長すぎるとロードが遅くなる
   - 本当に必要な情報だけを含める

3. **MCPと組み合わせる**
   - Skillsで方法を教え、MCPでアクセスを提供
   - 両者を組み合わせることで真の力を発揮

## おわりに

Claude Skillsは、**「専門知識の再利用」を解決する**効率的な仕組みです。

Skillsを使うことで、毎回同じ説明をする必要がなくなり、会話やプロジェクトをまたいで専門知識を活用できます。Prompts、Projects、MCP、Subagentsといった他のツールと組み合わせることで、個別のツールでは実現できない強力なワークフローを構築できます。

### 今すぐ始めるには

1. `.claude/skills/`ディレクトリを作成
2. 最初のSkillフォルダーを作成（例: `excel-expert`）
3. `SKILL.md`ファイルに説明と指示を記述
4. Claudeを使ってSkillが適切にロードされるか確認

### 今後の展望

Skillsはまだ進化中の機能です。今後、以下のような発展が期待されます:

- Skillsの共有マーケットプレイス
- より高度な動的ロード機構
- チーム内でのSkills管理機能

Skillsを使って、AIワークフローをさらに効率化していきましょう。
