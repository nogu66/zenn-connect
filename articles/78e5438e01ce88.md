## はじめに

「デザインからSwiftのコード実装まで、何度も行き来して時間がかかる...」
「UIの細かい調整を手動でコーディングするのが面倒...」

iOSアプリ開発において、デザインとコードの往復は避けられない作業ですが、時間がかかり非効率です。本記事では、**Figma Make**でデザインの大枠を作成し、それを**Claude Code**に渡してSwiftコードに変換させる手法を紹介します。

### この記事の対象読者

- iOSアプリ開発者で、デザインからコード化の効率を上げたい方
- Figmaでデザインを作成し、Swiftでの実装に悩んでいる方
- AI支援ツールを活用した開発フローに興味がある方

### この記事で分かること

- Figma Makeの基本的な使い方
- Claude Codeを使ったSwiftコード生成の実践方法
- デザインからコード化までのワークフロー
- 実際の画面デザインをSwiftUIに変換する具体例

### この記事で扱わないこと

- Figmaの詳細な操作方法
- SwiftUIの基礎文法
- 複雑なアニメーションやカスタムコンポーネントの実装

## 問題:デザインとコードの往復に時間がかかる

従来のiOSアプリ開発では、以下のような課題がありました:

1. **手動コーディングの手間**: Figmaのデザインを見ながら、色、サイズ、レイアウトを一つずつSwiftUIで書く
2. **デザイン変更の反映**: デザイン修正の度に、コードも手動で修正が必要
3. **認識の齟齬**: デザイナーの意図とエンジニアの実装が微妙にズレる
4. **初期画面の量産**: 特に新規アプリの初期段階で、多くの画面を実装する必要がある

これらの問題により、本来注力すべきビジネスロジックや機能開発に時間を割けないことが課題でした。

## 解決策:Figma Make × Claude Codeのワークフロー

Figma MakeとClaude Codeを組み合わせることで、デザインからSwiftコードへの変換を大幅に効率化できます。

### Figma Makeとは

Figma Makeは、Figmaのデザインからコードを生成する機能です。React、Vue、SwiftUIなど様々なフレームワークに対応しており、デザインの構造を理解してコードに変換します。

### Claude Codeとは

Claude Codeは、Anthropic社が提供するAI支援開発ツールです。自然言語でコードの生成、修正、リファクタリングを行えます。特に、デザインスクリーンショットやコードスニペットを渡して、実際に動作するコードに変換する能力に優れています。

### ワークフロー全体像

```
1. Figmaでデザイン作成
   ↓
2. Figma Makeでコード生成(HTML/CSS or SwiftUI)
   ↓
3. 生成コードとデザイン画像をClaude Codeに渡す
   ↓
4. Claude CodeがSwiftUIコードに変換・最適化
   ↓
5. 生成されたコードをXcodeプロジェクトに組み込み
   ↓
6. 微調整とビジネスロジックの実装
```

このワークフローにより、デザインからコード化までの時間を大幅に短縮できます。

## 実践:具体的な手順

それでは、実際にログイン画面を例に、Figma MakeとClaude Codeを使ってSwiftUIコードを生成する手順を解説します。

### Step 1: Figmaでデザインを作成

まず、Figmaで実装したい画面のデザインを作成します。ここでは、シンプルなログイン画面を例にします。

**デザインのポイント:**
- コンポーネントを明確に分ける(ボタン、テキストフィールドなど)
- 色やフォントサイズを一貫性を持たせる
- レイヤー名を分かりやすくつける(例: "LoginButton", "EmailTextField")

### Step 2: Figma Makeでコードを生成

1. Figmaでデザインを選択
2. 右側のパネルから「Code」タブを開く
3. Figma Makeを選択し、SwiftUIまたはHTMLを選択
4. 生成されたコードをコピー

:::message
Figma MakeのSwiftUI出力は参考レベルです。完璧なSwiftUIコードではないため、次のステップでClaude Codeに最適化してもらいます。
:::

### Step 3: デザインのスクリーンショットを取得

Figmaのデザイン画面全体のスクリーンショットを撮影します。これをClaude Codeへの入力として使用します。

### Step 4: Claude Codeにコード変換を依頼

Claude Codeを起動し、以下の情報を渡します:

```
以下のログイン画面をSwiftUIで実装してください。

[Figmaのスクリーンショット画像を添付]

参考コード(Figma Makeで生成):
[生成されたコードを貼り付け]

要件:
- SwiftUI 5.0を使用
- ダークモード対応
- アクセシビリティ対応
- ボタンタップ時のアクションは@Bindingで受け取る
- フォーム入力のバリデーション機能を含める
```

### Step 5: Claude Codeが生成したSwiftUIコード例

Claude Codeは以下のような実装可能なSwiftUIコードを生成してくれます:

```swift
import SwiftUI

struct LoginView: View {
    @State private var email: String = ""
    @State private var password: String = ""
    @State private var isPasswordVisible: Bool = false
    @State private var showError: Bool = false
    @State private var errorMessage: String = ""

    var onLogin: (String, String) -> Void

    var body: some View {
        VStack(spacing: 24) {
            // ロゴエリア
            Image(systemName: "lock.shield.fill")
                .resizable()
                .scaledToFit()
                .frame(width: 80, height: 80)
                .foregroundStyle(.blue)
                .padding(.top, 60)

            Text("ログイン")
                .font(.system(size: 32, weight: .bold))
                .padding(.bottom, 8)

            // メールアドレス入力
            VStack(alignment: .leading, spacing: 8) {
                Text("メールアドレス")
                    .font(.system(size: 14, weight: .medium))
                    .foregroundStyle(.secondary)

                TextField("example@email.com", text: $email)
                    .textFieldStyle(.roundedBorder)
                    .textInputAutocapitalization(.never)
                    .keyboardType(.emailAddress)
                    .autocorrectionDisabled()
            }

            // パスワード入力
            VStack(alignment: .leading, spacing: 8) {
                Text("パスワード")
                    .font(.system(size: 14, weight: .medium))
                    .foregroundStyle(.secondary)

                HStack {
                    if isPasswordVisible {
                        TextField("パスワードを入力", text: $password)
                    } else {
                        SecureField("パスワードを入力", text: $password)
                    }

                    Button(action: {
                        isPasswordVisible.toggle()
                    }) {
                        Image(systemName: isPasswordVisible ? "eye.slash.fill" : "eye.fill")
                            .foregroundStyle(.secondary)
                    }
                }
                .textFieldStyle(.roundedBorder)
            }

            // エラーメッセージ
            if showError {
                Text(errorMessage)
                    .font(.system(size: 14))
                    .foregroundStyle(.red)
                    .frame(maxWidth: .infinity, alignment: .leading)
            }

            // ログインボタン
            Button(action: {
                handleLogin()
            }) {
                Text("ログイン")
                    .font(.system(size: 16, weight: .semibold))
                    .foregroundStyle(.white)
                    .frame(maxWidth: .infinity)
                    .frame(height: 50)
                    .background(
                        RoundedRectangle(cornerRadius: 12)
                            .fill(isLoginEnabled ? Color.blue : Color.gray)
                    )
            }
            .disabled(!isLoginEnabled)

            // パスワードを忘れた場合
            Button(action: {
                // パスワードリセット処理
            }) {
                Text("パスワードを忘れた場合")
                    .font(.system(size: 14))
                    .foregroundStyle(.blue)
            }

            Spacer()
        }
        .padding(.horizontal, 24)
        .background(Color(.systemBackground))
    }

    private var isLoginEnabled: Bool {
        !email.isEmpty && !password.isEmpty && isValidEmail(email)
    }

    private func isValidEmail(_ email: String) -> Bool {
        let emailRegex = "[A-Z0-9a-z._%+-]+@[A-Za-z0-9.-]+\\.[A-Za-z]{2,64}"
        let emailPredicate = NSPredicate(format: "SELF MATCHES %@", emailRegex)
        return emailPredicate.evaluate(with: email)
    }

    private func handleLogin() {
        guard isValidEmail(email) else {
            errorMessage = "有効なメールアドレスを入力してください"
            showError = true
            return
        }

        guard password.count >= 6 else {
            errorMessage = "パスワードは6文字以上で入力してください"
            showError = true
            return
        }

        showError = false
        onLogin(email, password)
    }
}

#Preview {
    LoginView { email, password in
        print("Login attempted with \(email)")
    }
}
```

## 実際に使ってみた結果

### 開発時間の比較

| 作業内容 | 従来の方法 | Figma Make + Claude Code |
|---------|-----------|------------------------|
| デザイン確認 | 10分 | 5分 |
| コード実装 | 60分 | 10分 |
| 微調整 | 30分 | 15分 |
| **合計** | **100分** | **30分** |

実際のログイン画面実装において、**約70%の時間削減**を実現できました。

### メリット

1. **初期実装の高速化**: デザインからコードへの変換がほぼ自動化
2. **一貫性の向上**: デザインに忠実なコードが生成される
3. **学習効果**: 生成されたコードから実装パターンを学べる
4. **リファクタリング支援**: Claude Codeに改善を依頼できる

### 注意点

1. **完璧ではない**: 100%完璧なコードが生成されるわけではない
2. **レビュー必須**: 生成されたコードは必ず人間がレビューする
3. **複雑な画面は分割**: 複雑なデザインは、パーツごとに分けて変換すると精度が上がる
4. **デザインの明確さ**: Figmaのデザインが曖昧だと、コードも曖昧になる
