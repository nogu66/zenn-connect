このテンプレートは、Zenn technical article writerスキルで使用する標準的な記事構造です。

```yaml

title: "[問題]を[技術]で解決する"
emoji: "🛠️"
type: "tech"
topics: ["topic1", "topic2", "topic3"]
published: false
---
```

## 記事作成時の注意点

### 必須チェック項目

- [ ] タイトルに問題と解決策の技術の両方が含まれている
- [ ] 自己紹介セクションが含まれている
- [ ] はじめにで対象読者、スコープ、ベネフィットが明確に示されている
- [ ] 問題提起が共感できる具体的な内容になっている
- [ ] すべてのコード例がテスト済みで動作する
- [ ] What/Why/Howが各主要ポイントで説明されている
- [ ] おわりにで重要なポイントが要約されている
- [ ] 次のステップが実行可能で明確
- [ ] すべての専門用語が初出時に定義されている
- [ ] 一文一義が徹底されている
- [ ] 参考リンクに重要な情報源がすべて含まれている
- [ ] Frontmatterが完全で正確
- [ ] フォローCTAが含まれている

### 文体・トーン

- 専門的だが親しみやすい
- 読者の課題に共感的
- 自信を持ちつつも謙虚
- 励ましとサポートの姿勢

### よく使うフレーズ

**問題提起**:
- "こんな経験はありませんか?"
- "〜で困っていませんか?"

**解決策の紹介**:
- "[技術名]は、この問題を解決します。"
- "[技術名]を使えば、[問題]を[解決策]できます。"

**説明の移行**:
- "具体的には..."
- "つまり..."
- "要するに..."

**励まし**:
- "実際にやってみましょう"
- "早速試してみます"
